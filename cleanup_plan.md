# Repo Minimalization Plan (A/B/C/D)

This plan trims the repo down to the **minimum set of code/config/prompts/docs** needed to keep these workflows usable:

- **A**: 3D CLI: `python -m scripts.core.pipeline ...`
- **B**: 2D CLI: `python scripts/run_pipeline.py ...`
- **C**: P3D → Wan2.1 dataset + two-stage SFT training (cache → train)
- **D**: SDXL/Kohya training + monitoring

## Safety Rule (active training)

If Wan2.1 training is running (tracked by `logs/wan21_train_from_cache_p3d_pairs_from_images.nohup.pid`), **do not delete or move `scripts/`**.  
This repo currently has an active training process; only analysis is performed until it finishes.

## How the keep/delete set is derived

Generated by:

```bash
python scripts/maintenance/derive_minimal_keep_set.py
```

Output report:

- `outputs/maintenance/minimal_keep_set_report.json`

Current report totals:

- Keep set: **602** paths
- Delete candidates: **5645** paths
  - `scripts/**`: **469**
  - `docs/**`: **158**
  - `prompts/**`: **5018**

Rules used (high level):

- Start from the A/B/C/D entrypoints (and a small always-keep list).
- Expand dependencies via:
  - Python imports (including relative imports)
  - Shell references to repo-local paths
  - Config files under `configs/**` (kept by convention + any repo-local paths referenced inside configs)
  - Heuristic for subprocess target scripts built via `Path(__file__) / 'generic' / ... / 'tool.py'`
- Everything under `scripts/**`, `docs/**`, `prompts/**` **not in the computed keep set** becomes a delete candidate.

## Current size hot spots (before minimalization)

Large repo-local artifact directories have already been cleaned:

- `outputs/` (cleared)
- `logs/` (cleared except active Wan2.1 log + pid)

## Apply after training finishes

When the training PID file is no longer running, apply the plan with:

```bash
bash scripts/maintenance/apply_minimal_cleanup.sh
```

This script reads `outputs/maintenance/minimal_keep_set_report.json` and deletes the computed `delete_candidates`.
